📌 Project Overview

This project is a hybrid AI-powered Jira story point estimation system designed to simulate real-world Scrum Master decision-making.

Unlike basic ML systems, this solution integrates multiple intelligent layers to produce accurate, explainable, and adaptive estimations.

🧠 Core Intelligence Layers

  • Machine Learning (TF-IDF + classification)
  • Rule-based domain logic
  • Semantic similarity (past Jira learning)
  • Risk & uncertainty detection
  • Dependency analysis
  • Business priority handling
  • Confidence-based correction
  • LLM-based explanation

🎯 Goal

To build a system that behaves like an experienced Scrum Master by combining data, logic, and contextual reasoning.

🧰 Technologies Used

FastAPI Scikit-learn TF-IDF Rule Engine Semantic Search Groq LLM Jira API JSON Learning System

🧠 ADVANCED ESTIMATION FRAMEWORK (PROFESSOR-LEVEL)

This system goes beyond basic estimation and applies real Scrum Master thinking:

🔥 1. Historical Intelligence (Past Jira Analysis)

Key Question: “Have we done this before?”

  • Find similar past Jira tickets
  • Compare features, modules, keywords
  • Use actual story points (not predicted)

If similarity > 80% → Trust historical points more than AI

Example:

Past Issue: API Integration
Similarity: 0.85
Story Points: 8
→ Current estimation ≈ 8

🔥 2. Market / Business Context

  • Production issues
  • Revenue-impact features
  • Urgent sprint deadlines

Higher urgency → Higher story points

🔥 3. Technical + Risk + Uncertainty

  • Unknown requirements
  • New features / exploration
  • Migration complexity

🔥 4. Dependency Analysis

  • External APIs
  • Third-party systems
  • Cross-team dependencies

🔥 5. Team Context

  • Team experience level
  • Technology familiarity

New tech → More uncertainty → Higher points

🔥 6. Breakdown Thinking (Real Scrum)

  • Frontend effort
  • Backend effort
  • Integration effort
  • Testing effort

⚙️ FINAL COMPUTATION MODEL

Final Score =
(AI * 0.3) +
(Technical * 0.2) +
(Historical * 0.3) +
(Market * 0.1) +
(Uncertainty * 0.05) +
(Dependency * 0.05)

Final score is then mapped to Fibonacci scale: 

1, 2, 3, 5, 8, 13, 21
